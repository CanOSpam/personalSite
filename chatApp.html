LAN Chat Application

Chat App is a simple instant messenger application that uses TCP to share chat messages between many clients and one server. Messages are encrypted end-to-end with a simple encryption scheme that changes the encoding of the text. This application was written for a school project as an exercise to learn about the Linux networking library.

 

For this project the key technology was TCP. Without networking users would only be able to talk to themselves, which tends to get old pretty quickly. Our client-server architecture was fairly simple. The server regularly polled its socket for new messages. When the server received a message it would echo the message out to every other connected client. The clients could then decrypt the message and display it along with a timestamp.

 

The GUI framework Qt was also instrumental in this project, like many of our school projects. Qt enabled us to easily keep a visual list of connected clients. Using built in functionality we were also able to save a client's chatlog.

 

Overall, this app was one of our more successful projects of the term. We ran into some minor issues using the Linux networking library, but once those were resolved it was mostly smooth sailing. If I had had more time, I would have implemented an encryption key generation and sharing technique to increase security. Currently, the application uses a built in key, which defeats much of the purpose of the app. Alternatively I could have switched to a different encryption library, but other libraries I looked at had significantly more overhead.
